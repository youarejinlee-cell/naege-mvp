# Log to Letter QA Checklist

출시 전, 그리고 큰 변경 후마다 아래 항목을 실제 앱 화면에서 확인한다.

## 0. 테스트 환경

- [ ] iOS 시뮬레이터에서 앱이 정상 실행된다.
- [ ] 개발 빌드에서 앱을 껐다 켜도 마지막 상태가 유지된다.
- [ ] 테스트 계정으로 Google 로그인이 가능하다.
- [ ] 비로그인 상태에서도 기록 작성, 캘린더 확인, 설정 변경이 가능하다.
- [ ] 로그인 상태에서는 서버 동기화 완료 문구 또는 오류 문구가 명확히 보인다.

## 1. 첫 진입과 메뉴

- [ ] 앱 상단에 `Log to Letter` 로고와 메뉴 버튼이 보인다.
- [ ] 메뉴 버튼을 누르면 오른쪽 플로팅 메뉴가 열린다.
- [ ] 메뉴 바깥 영역을 누르면 메뉴가 닫힌다.
- [ ] 메뉴 우측 상단 `x` 버튼으로 메뉴가 닫힌다.
- [ ] 메뉴에 프로필 이미지와 사용자 이름이 보인다.
- [ ] `계정`, `설정`, `가이드` 메뉴로 각각 이동할 수 있다.

## 2. 기록 탭

- [ ] 제목은 `기록`으로 보인다.
- [ ] 서브타이틀은 두 줄로 보인다.
  - `지금 이 순간의 생각과 감정을 솔직하게 남겨봐.`
  - `그 기록이 미래의 너에게 의미로 돌아올 거야.`
- [ ] 텍스트 입력란의 플레이스홀더가 랜덤 문구 중 하나로 보인다.
- [ ] 한국어 입력 시 자모음이 과도하게 분리되지 않는다.
- [ ] 힌트 버튼을 누르면 자연스러운 문장 시작 문구가 입력된다.
- [ ] 생각, 감정, 에너지를 모두 선택해야 `남기기`가 동작한다.
- [ ] 필수값이 빠진 경우 저장되지 않는다.
- [ ] 정상 저장 후 `남기기 완료` 팝업이 보인다.
- [ ] 저장 직후 캘린더와 기간별 기록에 해당 기록이 반영된다.
- [ ] 에너지 선택 UI가 현재 팔레트 색상을 반영한다.
- [ ] 긍정/중간/부정 감정 목록이 접힘/펼침 상태에서 잘 보인다.

## 3. 캘린더 탭

- [ ] 오늘 날짜가 정상 표시된다.
- [ ] 오늘 이후 날짜는 비활성화되어 선택할 수 없다.
- [ ] 날짜를 누르면 해당 날짜가 선택된다.
- [ ] 선택된 날짜의 테두리와 숫자가 정상 표시된다.
- [ ] 기록이 없는 날짜는 기록이 있는 날짜와 명확히 구분된다.
- [ ] 날짜별 보기에서 선택한 날짜의 기록 리스트가 보인다.
- [ ] 기록이 없을 때 `남긴 기록이 없어` 문구가 보인다.
- [ ] 하루 기록 리스트의 날짜 제목은 `2026년 6월 6일` 형식으로 보인다.
- [ ] 하루 기록 카드의 시간은 `오전 9시 43분`, `오후 5시 55분` 형식으로 보인다.
- [ ] 기록 카드 상단에 감정과 에너지가 나란히 보인다.
- [ ] `이달 요약`이 캘린더와 같은 박스 안에 보인다.
- [ ] `이달 요약`의 가장 많았던 감정이 버튼 형태로 최대 3개 보인다.
- [ ] 감정이 없으면 `-`가 보인다.
- [ ] 에너지 좋았던 날/낮았던 날은 일자별 평균 기준으로 산출된다.
- [ ] 에너지 좋았던 날/낮았던 날 날짜는 `2026년 6월 6일` 형식이다.

## 4. 기간별 기록

- [ ] `기간별` 탭으로 전환할 수 있다.
- [ ] `1주`, `1개월`, `3개월`, `직접 입력`이 보인다.
- [ ] 직접 입력 모드에서 시작일과 종료일을 각각 입력할 수 있다.
- [ ] `확인` 버튼을 누르기 전에는 결과 목록이 바뀌지 않는다.
- [ ] `확인` 버튼을 누르면 선택 기간의 기록만 보인다.
- [ ] 기간 제목의 날짜 형식은 `2026년 6월 6일` 형식이다.
- [ ] 최신순/오래된순 필터가 동작한다.
- [ ] 요약은 카드 리스트 상단에 보인다.
- [ ] 선택 버튼을 누르면 카드 선택 모드가 켜진다.
- [ ] 선택 모드에서 `취소`, `삭제`가 같은 액션 영역에 나타난다.
- [ ] 삭제를 누르면 `정말로 삭제하시겠습니까?` 확인 카드가 뜬다.
- [ ] `아니오`를 누르면 삭제되지 않는다.
- [ ] `네`를 누르면 선택한 기록이 삭제되고 캘린더/요약도 갱신된다.

## 5. 편지보관함

- [ ] 탭 이름은 `편지보관함`으로 보인다.
- [ ] 월별 편지 목록이 먼저 보인다.
- [ ] 편지 아이콘은 `letter-archive-icon.png` 이미지로 보인다.
- [ ] 편지 아이콘의 봉투 실루엣이 잘리지 않고 온전히 보인다.
- [ ] 편지 요약 제목이 9자를 넘으면 `...`으로 줄어든다.
- [ ] 편지 아이콘을 누르면 편지 상세 화면으로 이동한다.
- [ ] 상세 화면 상단 문구는 `편지보관함`이다.
- [ ] `← 목록으로`를 누르면 목록으로 돌아간다.
- [ ] 편지 제목 아래에 받은 날짜와 기록 기간이 보인다.
- [ ] 기록 기간은 `2026년 6월 6일` 형식으로 보인다.
- [ ] 긴 편지는 좌우 슬라이드 페이지로 나뉜다.
- [ ] 페이지 표시 점이 현재 페이지를 반영한다.
- [ ] `추신`을 입력하고 저장할 수 있다.
- [ ] `편지 내용 이미지로 저장` 버튼이 보인다.
- [ ] 이미지 저장 권한 요청이 정상 동작한다.
- [ ] 저장 성공/실패 메시지가 보인다.

## 6. 편지지 디자인

- [ ] 설정의 `편지지 디자인`에 `무지`, `편지지 테두리`, `제목+내용 분리`, `클로버 무늬`가 보인다.
- [ ] `제목 형광펜` 옵션은 보이지 않는다.
- [ ] `무지`는 기본 흰 편지지로 보인다.
- [ ] `편지지 테두리`는 현재 테마색 테두리를 반영한다.
- [ ] `제목+내용 분리`는 본문만 둥근 박스 안에 들어간다.
- [ ] `제목+내용 분리`에서 제목, 날짜, 앱 브랜드 영역은 본문 박스 밖에 있다.
- [ ] `클로버 무늬` 설명은 `세잎클로버 틈바구니에 숨어 있는 네잎클로버 하나`로 보인다.
- [ ] `클로버 무늬`는 편지지 디자인 옵션의 가장 아래에 있다.
- [ ] `클로버 무늬` 편지지의 클로버 패턴은 현재 테마색을 반영한다.
- [ ] `클로버 무늬` 편지지에는 세잎클로버들 사이에 네잎클로버 하나가 숨어 있다.

## 7. 알림 설정

- [ ] 탭 이름은 `알림` 또는 현재 앱 내 알림 탭 명칭과 일치한다.
- [ ] 알림 설정 제목 오른쪽에 on/off 스위치가 있다.
- [ ] 알림이 꺼져 있으면 요약에 `알림이 설정되어 있지 않아`가 보인다.
- [ ] 알림이 켜져 있으면 요약에 설정값이 보인다.
- [ ] 요약에는 하루 또는 일주일에 몇 번의 기록을 할 수 있는지 보인다.
- [ ] `변경` 버튼을 누르면 설정 편집 화면으로 들어간다.
- [ ] 간격 반복 모드와 특정 시간 모드를 전환할 수 있다.
- [ ] 간격 반복 모드는 10분부터 120분까지 5분 단위로 선택된다.
- [ ] 특정 시간 모드는 월~일 복수 선택과 시간 추가/삭제가 가능하다.
- [ ] 시작 시간이 방해금지 시간 안에 있으면 경고 문구가 두 줄로 보인다.
- [ ] 저장 버튼 위 시뮬레이션 박스에 예상 기록 횟수가 보인다.
- [ ] 저장 후 요약 화면으로 돌아간다.
- [ ] 저장 실패 시 서버/네트워크 오류 문구가 보인다.

## 8. 설정

- [ ] 메뉴의 `설정`은 알림 탭이 아니라 별도 설정 화면으로 이동한다.
- [ ] 컬러 테마는 빨강, 노랑, 초록, 파랑이 보인다.
- [ ] 컬러 테마 변경 시 하단 탭, 버튼, 주요 텍스트 색이 함께 바뀐다.
- [ ] 앱을 껐다 켜도 마지막 컬러 테마가 유지된다.
- [ ] 에너지 팔레트는 소프트/비비드 옵션이 보인다.
- [ ] 에너지 팔레트는 클로버 모양 컬러칩으로 보인다.
- [ ] 앱을 껐다 켜도 마지막 에너지 팔레트가 유지된다.
- [ ] 캘린더 에너지 표시 옵션 3개가 보인다.
- [ ] 앱을 껐다 켜도 마지막 캘린더 에너지 표시 설정이 유지된다.

## 9. 계정과 동기화

- [ ] 메뉴에서 `계정` 화면으로 이동할 수 있다.
- [ ] 계정 화면에 이메일 주소가 보인다.
- [ ] `내 기록과 편지 완전 삭제` 문구가 보인다.
- [ ] 로그아웃이 동작한다.
- [ ] 로그아웃 후 비로그인 상태 UI가 보인다.
- [ ] 다시 로그인하면 서버에 저장된 기록/편지가 불러와진다.
- [ ] 기록 저장 후 Supabase `entries`에 저장된다.
- [ ] 알림 설정 저장 후 Supabase `notification_settings`에 저장된다.
- [ ] 앱 설정 저장 후 Supabase `app_settings`에 저장된다.
- [ ] RLS 오류 없이 읽기/쓰기/삭제가 가능하다.

## 10. 가이드와 테스트 콘솔

- [ ] 메뉴에서 `가이드` 화면으로 이동할 수 있다.
- [ ] 가이드는 실제 탭 이름과 사용 순서를 설명한다.
- [ ] 테스트 콘솔은 개발/프리뷰 환경에서만 접근 가능하다.
- [ ] 운영 빌드에서는 테스트 콘솔이 노출되지 않는다.
- [ ] 테스트 콘솔에서 테스트 날짜를 설정할 수 있다.
- [ ] 테스트 날짜로 샘플 기록을 넣으면 해당 날짜 캘린더에 반영된다.

## 11. 회귀 테스트

- [ ] 새 기록 작성 후 캘린더 날짜 색/요약/기간별 목록이 모두 갱신된다.
- [ ] 기록 삭제 후 편지보관함의 이미 생성된 편지 내용은 바뀌지 않는다.
- [ ] 첫 기록 후 D+7 기준 편지 생성 규칙이 유지된다.
- [ ] 이미 생성된 편지는 이후 기록 추가로 내용이 바뀌지 않는다.
- [ ] 앱을 껐다 켜도 기록, 설정, 편지, 추신이 유지된다.
- [ ] 로그인 상태와 비로그인 상태를 오가도 데이터가 의도대로 유지된다.

## 12. 출시 전 최종 확인

- [ ] `npx tsc --noEmit`이 통과한다.
- [ ] iOS 시뮬레이터에서 주요 화면 스크롤이 막히지 않는다.
- [ ] 앱 아이콘이 `assets/app-icon.png`로 반영된다.
- [ ] 편지보관함 아이콘이 `assets/letter-archive-icon.png`로 반영된다.
- [ ] 앱 이름 `Log to Letter`가 주요 화면과 저장 이미지에 일관되게 보인다.
- [ ] 개인정보 처리방침에 기록, 편지, 알림, 이미지 저장, 로그인, AI 사용 범위가 반영되어 있다.
- [ ] App Store 제출용 스크린샷과 설명 문구가 준비되어 있다.
